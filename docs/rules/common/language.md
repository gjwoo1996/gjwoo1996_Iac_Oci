# 언어 사용 규칙

**규칙 ID:** `common/language`  
**적용 대상:** 이 프로젝트에서 작업하는 모든 AI  
**적용 범위:** 모든 답변, 문서 작성, 코드 주석, 커밋 메시지 등

---

## 원칙

### 1. 한국어 우선 원칙

모든 답변과 문서는 **한국어**로 작성한다.

적용 범위:
- AI의 답변 및 설명
- 문서 (Markdown, README 등)
- 코드 주석
- 커밋 메시지
- PR 설명 및 이슈 내용

### 2. 언어 전환 고지 원칙

다른 언어로 작성하는 것이 **AI의 성능·정확도·이해도** 측면에서 명백히 유리한 경우, 사용자에게 이유를 먼저 고지한 후 해당 언어를 사용할 수 있다.

**고지 형식 예시:**

> "이 부분은 [이유]로 인해 영어로 작성하는 것이 더 정확합니다. 영어로 작성하겠습니다."

**언어 전환이 허용되는 예시 상황:**
- 기술 용어·라이브러리명·API 명세처럼 원어 그대로 표기해야 의미가 명확한 경우
- 특정 언어로만 작성된 공식 문서를 그대로 인용해야 하는 경우
- AI 모델의 특성상 영어 프롬프트가 결과 품질에 직접적으로 영향을 주는 경우

**허용되지 않는 경우:**
- 단순히 작성이 편하다는 이유
- 고지 없이 언어를 전환하는 행위

---

## 준수 체크리스트

AI는 작업 전 다음을 확인한다:

- [ ] 답변을 한국어로 작성하고 있는가?
- [ ] 한국어가 아닌 언어를 사용해야 한다면, 이유를 사용자에게 고지했는가?
- [ ] 기술 용어는 원어를 병기하되 설명은 한국어로 작성했는가?
